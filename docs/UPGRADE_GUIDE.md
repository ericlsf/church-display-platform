# Upgrade guide

1. Create a platform backup.
2. Confirm the Git working tree is clean.
3. Review release notes and migration instructions.
4. Validate the release artifact and checksum.
5. Dry-run on one test display.
6. Put the fleet in Maintenance Mode for disruptive upgrades.
7. Deploy to one test display and verify heartbeat, preview, sync, and playback.
8. Deploy to additional groups or sites in controlled batches.
9. Exit Maintenance Mode after verification.
10. Keep the rollback tag and backup until the release has been stable through normal operation.
